# 查课结果上报微信小程序实施方案

## 一、总体架构

- **前端**：微信小程序原生（老师端 + 管理端共用一个小程序，按角色显示页面）。
- **后端**：微信云开发 Cloud Functions。
- **数据库**：云数据库（3 个集合）。
- **文件存储**：云存储（现场照片）。

## 二、页面与路由

### 1) 公共页面
- `pages/auth/index`：登录、首次绑定身份。
- `pages/profile/index`：个人信息、退出、异常说明入口。

### 2) 老师端
- `pages/home/index`：今日待查数/已提交数、快捷入口。
- `pages/form/index`：问卷式填报。
- `pages/records/index`：我的记录 + 状态筛选。

### 3) 管理端
- `pages/admin-dashboard/index`：今日概览（提交老师数、异常数、待跟进数）。
- `pages/admin-list/index`：筛选 + 列表。
- `pages/admin-detail/index`：详情 + 跟进备注。
- `pages/admin-stats/index`：按天/老师/课程/校区统计。

## 三、核心流程

### 老师提交流程
1. 登录后读取角色。
2. 进入填报页，默认日期=今天、查课老师=本人。
3. 选择课程模板（自动带出校区/班级/课程/授课老师/节次）。
4. 完成问卷并上传图片。
5. 点击“提交”调用 `submitInspection`。
6. 跳转“我的记录”查看结果。

### 管理员流程
1. 进入总览页调用 `getAdminOverview`。
2. 通过列表页按日期/老师/校区/异常筛选。
3. 查看详情并标记“已跟进”，记录跟进备注。
4. 需要汇报时调用 `exportRecords` 导出。

## 四、阶段拆分

### 第一阶段（1~2 周）
- 登录/角色识别
- 填报+图片上传
- 我的记录
- 管理列表+异常筛选

### 第二阶段（1 周）
- 管理总览统计
- 导出 Excel
- 未提交提醒（订阅消息）

### 第三阶段（按需）
- 自动日报/周报
- 多管理员协作权限
- 课程预排与智能推荐模板

## 五、验收标准（MVP）

- 10 名老师可正常登录并提交。
- 单条上报平均填写耗时 ≤ 1 分钟。
- 管理员可筛出任意日期异常记录并查看图片。
- 当日提交率、异常数、待跟进数可在首页展示。
